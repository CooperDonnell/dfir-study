## Baseline answers

### Hashing

Initial answer: hashing establishes integrity and can show whether something
changed.

Refinement: matching hashes strongly support that the bytes have not changed
between two points in time. A hash alone does not prove that collection was
correct, that the original evidence was authentic, who possessed it, or what
the evidence means.

### IOC, artifact, and finding

- **Artifact:** Data created or modified through system or user activity, such
  as an event log, Registry value, Prefetch file, browser record, LNK file, or
  process in memory.
- **Indicator/IOC:** An observable associated with potentially malicious
  activity, such as a hash, domain, IP address, filename, Registry path, or
  behavioral pattern. An indicator is not automatically proof of compromise.
- **Finding:** An analyst's supported conclusion drawn from evidence, with
  appropriate confidence and limitations.

Completed a crackme exercise by examining defined strings, following relevant

### HeaderHunter

HeaderHunter reads headers, format structures, and exposed metadata—including
magic values—to identify supported file/container types and visible encryption
information. Testing included many file types across macOS, Windows, and Linux.

The Python implementation was generated by Codex. Public descriptions and
interview answers must accurately distinguish:

- The investigative problem and requirements personally identified
- Testing and validation personally performed
- Design decisions personally directed or reviewed
- Code generated by AI
- Code personally understood, modified, or verified

Do not claim sole authorship of code that cannot be explained.
